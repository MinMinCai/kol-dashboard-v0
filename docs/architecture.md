# 系統架構設計

## 1) KOL 資料庫

### 主要資料
- 基本資料：姓名/暱稱、地區、語言、聯絡方式、類別標籤
- 社群資料：平台、粉絲數、互動率、平均觀看、受眾輪廓
- 商務資料：報價區間、授權條件、可合作檔期
- 合作歷史：品牌、檔期、內容型態、成果指標、評價

### 主要操作
- KOL 建立、編輯、封存
- 多條件搜尋（平台、類別、粉絲區間、互動率）
- 依過往成效排序推薦

## 2) 提案專案系統

### 流程節點
- `draft` 草稿
- `internal_review` 內部審核
- `sent_to_client` 送客戶
- `client_feedback` 客戶反饋
- `revised` 修訂
- `won/lost` 成功或失敗

### 記錄重點
- 客戶偏好（風格、預算、避雷項）
- 每輪反饋與修正版本
- 中標/失標原因

## 3) 委刊單管理

### 核心欄位
- 委刊單編號、對應提案、品牌/客戶、KOL、預算
- 檔期、交付內容、審稿節點、付款節點
- 合約狀態、發票狀態、交付完成率

### 狀態流
- `created -> signed -> in_execution -> delivered -> settled -> closed`

## 4) AI 輔助

### 功能
- 智能搜尋：自然語言找 KOL（例：美妝、女性 18-24、互動率 > 4%）
- 成效辨識：找出高 ROI 合作模式與風險特徵
- 報告生成：自動生成提案摘要、月報、結案報告

### 技術做法
- 結構化資料查詢（SQL）+ 向量檢索（embedding）
- 使用任務佇列非同步產生報告
- 報告輸出 Markdown/PDF

## 5) 成效追蹤

### KPI 維度
- 曝光：impressions, reach, views
- 互動：likes, comments, shares, saves, engagement_rate
- 轉換：clicks, CTR, leads, purchases, ROAS
- 品質：準時率、素材品質評分、客戶滿意度

### 關鍵設計
- 每次合作都要有「事前目標」與「事後實績」
- 支援平台拆分（IG/YT/TikTok）與總覽彙總
- 支援同檔多波次分析
